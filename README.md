# TeleMT Proxy Manager

## 📖 Описание
Скрипт для автоматической установки и управления MTProto прокси для Telegram на базе TeleMT (Rust). Поддерживает три режима работы с возможностью запуска нескольких контейнеров одновременно.

## 🚀 Режимы прокси

| Режим | Описание | Шифрование | Маскировка | Когда использовать |
|-------|----------|------------|------------|--------------------|
| CLASSIC | Обычный MTProto | ❌ Нет | ❌ Нет | Только если другие режимы не работают |
| DD | Secure MTProto | ✅ Да | ❌ Нет | Повседневное использование (рекомендуемый) |
| EE | FakeTLS | ✅ Да | ✅ HTTPS | Для стран с жесткой цензурой |

## 🛡️ Системные требования
- ОС: Ubuntu 20.04/22.04/24.04, Debian 11/12
- Права: root доступ
- Docker: устанавливается автоматически

## 🚀 Быстрый запуск

```bash
# Скачать скрипт
# Запустить с правами root
sudo bash deploy_telemt_proxy.sh
```
## 📋 Возможности
- ✅ Установка всех трех режимов на одном сервере
- ✅ Автоматическое определение свободных портов
- ✅ Генерация случайных секретов
- ✅ Вставка своих секретов
- ✅ Использование своих портов для proxy
- ✅ Поддержка переустановки существующих контейнеров
- ✅ Автоматическое открытие портов в UFW
- ✅ Готовые ссылки для Telegram

## 🔧 Что делает скрипт
1. Проверяет права root
2. Устанавливает зависимости (curl, openssl, xxd, iproute2)
3. Устанавливает Docker, если отсутствует
4. Проверяет существующие контейнеры
5. Запрашивает режим и параметры
6. Создает конфигурацию и docker-compose
7. Запускает контейнер
8. Открывает порт в фаерволе
9. Выводит ссылку для подключения

## 📂 Структура установки

| Режим   | Директория            | Контейнер        | Порт по умолчанию |
|---------|-----------------------|------------------|-------------------|
| CLASSIC | /opt/telemt-classic   | telemt-classic   | 8443              |
| DD      | /opt/telemt-dd        | telemt-dd        | 8444              |
| EE      | /opt/telemt-ee        | telemt-ee        | 8445              |

## 📁 Структура файлов
```
/opt/
├── telemt-classic/          # CLASSIC режим
│   ├── config/telemt.toml   # Конфигурация
│   ├── docker-compose.yml   # Docker Compose
│   ├── secret.txt           # Секрет
│   └── link.txt             # Ссылка для подключения
├── telemt-dd/               # DD режим
└── telemt-ee/               # EE режим
```

## 📝 Команды управления
```bash
# Просмотр логов
docker logs telemt-<mode> -f
# Остановка прокси
docker stop telemt-<mode>
# Запуск прокси
docker start telemt-<mode>
# Перезапуск
docker restart telemt-<mode>
# Удаление контейнера
docker rm -f telemt-<mode>
```
## 🔗 Подключение в Telegram
После установки скрипт выводит ссылку вида:
> tg://proxy?server=IP&port=PORT&secret=SECRET

Просто нажмите на нее в Telegram для автоматической настройки.

## ⚙️ Ручная настройка
Если ссылка не открывается:
1. Откройте Настройки Telegram
2. Выберите Продвинутые настройки → Тип соединения
3. Выберите Прокси → Добавить прокси
4. Тип: MTProto
5. Введите сервер, порт и секрет

## 🐛 Устранение проблем
```bash
# Проверить что контейнер запущен
docker ps | grep telemt
# Проверить порт
ss -tuln | grep ПОРТ
# Посмотреть логи
docker logs telemt-<mode> --tail 50
```
### Проблема: "pull rate limit exceeded"
Решение: Авторизуйтесь в Docker Hub:
```bash 
docker login
# Введите логин и пароль от hub.docker.com
docker compose up -d
```
### Получить правильную ссылку
```bash
# Через API (самый точный способ)
curl -s http://127.0.0.1:9092/v1/users | jq -r '.data[0].links.secure[0]'
```

## ❓ Частые вопросы
1. Какой режим выбрать?
Рекомендуется DD для повседневного использования. EE используйте при блокировках.
2. Можно ли установить все три режима?
Да, скрипт поддерживает установку всех режимов на одном сервере с разными портами.
3. Как посмотреть существующие прокси?
Запустите скрипт еще раз и выберите пункт "Показать ссылки".
4. Как удалить прокси?
```bash
# Удалить все прокси
sudo ./deploy_telemt_proxy.sh
# Выберите пункт 2 → 5 (Все режимы)
```
```bash
docker rm -f telemt-<mode>
rm -rf /opt/telemt-<mode>
```
